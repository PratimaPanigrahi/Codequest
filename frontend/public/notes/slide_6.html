Working with Data Types

Understanding Python's built-in data types

Data types are classifications that specify which type of value a variable can hold. Python has several built-in data types that are essential for programming.

Understanding data types is crucial because:

  • Different operations work with different data types
  • Memory allocation depends on the data type
  • Type-specific methods are available for each data type
  • Python is dynamically typed, but type matters for operations
Did you know? Python automatically determines the data type when you assign a value to a variable.
# Python determines data types automatically
name = "Alice" # str (string)
age = 25 # int (integer)
height = 5.7 # float (floating point)
is_student = True # bool (boolean)

# Check data type with type() function
print(type(name)) # <class 'str'>
print(type(age)) # <class 'int'>

Numeric Data Types

Working with numbers in Python

Python has three main numeric data types:

  • int - Integer numbers (whole numbers)
  • float - Floating-point numbers (decimal numbers)
  • complex - Complex numbers (real + imaginary parts)

Python automatically handles the conversion between int and float when necessary.

Python 3 Fact: There's no limit to how long an integer value can be, except for available memory.
# Integer examples
x = 10
y = -5
z = 1000000

# Float examples
a = 3.14
b = -2.5
c = 1.0

# Complex examples
d = 3 + 4j
e = complex(2, -3)

# Arithmetic operations
result1 = 10 + 3.5 # Result is float: 13.5
result2 = 7 / 2 # Result is float: 3.5
result3 = 7 // 2 # Result is int: 3 (floor division)

Strings

Working with text in Python

Strings are sequences of characters used to represent text. In Python, strings are immutable and can be defined using single, double, or triple quotes.

Key string features:

  • Indexing and slicing operations
  • Multiple built-in methods for manipulation
  • Support for escape characters
  • String formatting capabilities
Fun Fact: Python strings are immutable, meaning once created, they cannot be changed.
# String creation
s1 = "Hello"
s2 = 'World'
s3 = '''Multi-line
string'''

# String operations
greeting = s1 + " " + s2 # Concatenation
print(greeting[0]) # Indexing: 'H'
print(greeting[0:5]) # Slicing: 'Hello'

# String methods
print(greeting.upper())
print(greeting.lower())
print(len(greeting)) # Length: 11

Lists

Ordered, mutable collections

Lists are ordered, mutable collections that can hold items of different data types. They are one of Python's most versatile data structures.

List characteristics:

  • Ordered collection of items
  • Mutable (can be changed after creation)
  • Can contain mixed data types
  • Support indexing and slicing
Did you know? Lists in Python are similar to arrays in other languages but more flexible.
# List creation
fruits = ["apple", "banana", "cherry"]
mixed_list = [1, "hello", 3.14, True]
numbers = list(range(1, 6)) # [1, 2, 3, 4, 5]

# List operations
fruits.append("orange") # Add item
fruits[1] = "blueberry" # Modify item
removed = fruits.pop() # Remove last item

# List slicing
print(numbers[1:3]) # [2, 3]
print(numbers[:3]) # [1, 2, 3]
print(numbers[3:]) # [4, 5]

Tuples and Sets

Immutable collections and unique elements

Tuples are ordered, immutable collections. Once created, their elements cannot be changed.

Sets are unordered collections of unique elements. They don't allow duplicates.

Performance Tip: Sets are optimized for membership testing (checking if an element exists).
# Tuple examples
coordinates = (10, 20)
colors = ("red", "green", "blue")
# coordinates[0] = 5 # This would cause an error!

# Set examples
unique_numbers = {1, 2, 3, 2, 1} # Becomes {1, 2, 3}
fruits_set = {"apple", "banana", "cherry"}
fruits_set.add("orange")
fruits_set.discard("banana")

Dictionaries

Key-value pairs for data organization

Dictionaries are unordered collections of key-value pairs. They are optimized for retrieving values when you know the key.

Dictionary characteristics:

  • Unordered collection of key-value pairs
  • Keys must be unique and immutable
  • Values can be of any data type
  • Fast lookup by key
Python Tip: Dictionaries are fundamental to Python and used extensively in the language itself.
# Dictionary creation
person = {
  "name": "Alice",
  "age": 25,
  "city": "New York"
}

# Accessing values
print(person["name"]) # "Alice"
print(person.get("age")) # 25

# Modifying dictionaries
person["age"] = 26
person["job"] = "Engineer"
del person["city"]

Type Conversion

Converting between different data types

Python provides functions to convert between different data types. This is useful when you need to perform operations that require specific types.

Common conversion functions:

  • int() - Converts to integer
  • float() - Converts to float
  • str() - Converts to string
  • list() - Converts to list
  • tuple() - Converts to tuple
  • set() - Converts to set
Note: Not all conversions are possible. Trying to convert inappropriate values will raise errors.
# Type conversion examples
num_str = "123"
num_int = int(num_str) # Convert to int

price = 19.99
price_str = str(price) # Convert to string

numbers = [1, 2, 3, 2, 1]
unique_nums = set(numbers) # Convert to set

Practical Applications

Using data types in real-world scenarios

Different data types are suited for different tasks. Choosing the right one can make your code more efficient and readable.

Common use cases:

  • Lists - When you need an ordered collection that may change
  • Tuples - For fixed collections of related items
  • Dictionaries - When you need to associate keys with values
  • Sets - For storing unique elements and membership testing
  • Strings - For all text processing tasks
Best Practice: Use the simplest data type that meets your needs. This makes code easier to understand and maintain.
# Real-world example: Student database
students = [
  {"name": "Alice", "grades": [85, 92, 78]},
  {"name": "Bob", "grades": [76, 88, 95]},
  {"name": "Charlie", "grades": [90, 87, 93]}
]

# Calculate average grade for each student
for student in students:
  name = student["name"]
  grades = student["grades"]
  avg_grade = sum(grades) / len(grades)
  print(f"{name}'s average: {avg_grade:.1f}")
