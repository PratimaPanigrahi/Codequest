Object-Oriented Programming

Understanding OOP concepts in Python

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects" which can contain data and code to manipulate that data.

OOP helps in organizing complex programs by:

  • Breaking problems into smaller, manageable objects
  • Encouraging code reuse through inheritance
  • Protecting data through encapsulation
  • Allowing flexible code through polymorphism
Did you know? Python supports multiple programming paradigms, including procedural, functional, and object-oriented programming.
# Simple class definition
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def bark(self):
        print(f"{self.name} says woof!")

# Creating objects (instances)
my_dog = Dog("Rex", 3)
my_dog.bark() # Output: Rex says woof!

Classes and Objects

The building blocks of OOP

A class is a blueprint for creating objects. It defines the attributes and methods that the objects will have.

An object is an instance of a class. It has actual values for the attributes defined in the class.

Class

Blueprint or template

Object

Instance of a class

Python Fact: Everything in Python is an object, including integers, strings, functions, and classes themselves.
# Defining a class
class Car:
    # Class attribute (shared by all instances)
    wheels = 4

    # Constructor method
    def __init__(self, brand, model, year):
        # Instance attributes
        self.brand = brand
        self.model = model
        self.year = year
        self.mileage = 0

    def drive(self, miles):
        self.mileage += miles
        print(f"Driving {miles} miles. Total mileage: {self.mileage}")

# Creating objects
car1 = Car("Toyota", "Camry", 2020)
car2 = Car("Honda", "Civic", 2022)

car1.drive(100)
print(car1.brand) # Toyota
print(Car.wheels) # 4 (class attribute)

Inheritance

Creating new classes from existing ones

Inheritance allows a class (child) to inherit attributes and methods from another class (parent). This promotes code reuse and establishes relationships between classes.

Types of inheritance in Python:

  • Single inheritance: A child class inherits from one parent class
  • Multiple inheritance: A child class inherits from multiple parent classes
  • Multilevel inheritance: A child class inherits from a parent which itself inherits from another class
Python Feature: Python supports multiple inheritance, allowing a class to inherit from multiple parent classes.
# Parent class
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        print(f"{self.name} makes a sound")

# Child class inheriting from Animal
class Dog(Animal):
    def speak(self):
        print(f"{self.name} says woof!")

# Another child class
class Cat(Animal):
    def speak(self):
        print(f"{self.name} says meow!")

# Using the classes
animal = Animal("Generic Animal")
dog = Dog("Rex")
cat = Cat("Whiskers")

animal.speak() # Generic Animal makes a sound
dog.speak() # Rex says woof!
cat.speak() # Whiskers says meow!

Encapsulation

Data hiding and access control

Encapsulation is the concept of bundling data (attributes) and methods that operate on that data within a single unit (class), and restricting direct access to some of the object's components.

Access modifiers in Python:

  • Public: Accessible from anywhere (default)
  • Protected: Accessible within the class and its subclasses (prefix with _)
  • Private: Accessible only within the class (prefix with __)
Python Convention: Python doesn't enforce access restrictions strictly. It's more about conventions and indicating intent.
# Example of encapsulation
class BankAccount:
    def __init__(self, account_holder, balance):
        self.account_holder = account_holder # Public
        self._account_number = "12345" # Protected
        self.__balance = balance # Private

    # Public method to access private attribute
    def get_balance(self):
        return self.__balance

    # Public method to modify private attribute
    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            print(f"Deposited ${amount}. New balance: ${self.__balance}")

    def withdraw(self, amount):
        if 0 < amount <= self.__balance:
            self.__balance -= amount
            print(f"Withdrew ${amount}. New balance: ${self.__balance}")

# Using the class
account = BankAccount("Alice", 1000)
print(account.account_holder) # Alice (public)
print(account.get_balance()) # 1000 (via method)
account.deposit(500)
# account.__balance # This would cause an error

Polymorphism

One interface, multiple implementations

Polymorphism allows objects of different classes to be treated as objects of a common superclass. It enables a single interface to represent different underlying forms (data types).

Types of polymorphism in Python:

  • Method overriding: Child class provides a specific implementation of a method already defined in its parent class
  • Method overloading: Python doesn't support traditional method overloading, but we can achieve similar functionality with default parameters
  • Duck typing: "If it walks like a duck and quacks like a duck, it must be a duck" - Python's approach to polymorphism
Python Philosophy: Python uses duck typing, focusing on an object's behavior rather than its type.
# Polymorphism with inheritance
class Shape:
    def area(self):
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height

    def area(self):
        return self.width * self.height

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14 * self.radius ** 2

# Duck typing example
class Triangle:
    def __init__(self, base, height):
        self.base = base
        self.height = height

    def area(self):
        return 0.5 * self.base * self.height

# Function that uses polymorphism
def print_area(shape):
    print(f"Area: {shape.area()}")

# Using different objects with the same interface
shapes = [Rectangle(4, 5), Circle(3), Triangle(6, 4)]
for shape in shapes:
    print_area(shape)

Special Methods

Magic methods in Python classes

Special methods (also called magic or dunder methods) allow you to define how objects of your class behave with Python's built-in operations.

Common special methods:

  • __init__: Constructor, called when creating an instance
  • __str__: Returns a string representation of the object
  • __len__: Defines behavior for the len() function
  • __add__: Defines behavior for the + operator
  • __eq__: Defines behavior for the == operator
Python Tip: Special methods are what make Python's syntax so expressive and intuitive for custom classes.
# Class with special methods
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        return f"Vector({self.x}, {self.y})"

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __eq__(self, other):
        return self.x == other.x and self.y == other.y

    def __len__(self):
        return 2 # Vectors always have 2 components

# Using the Vector class
v1 = Vector(2, 3)
v2 = Vector(1, 4)

print(v1) # Vector(2, 3) - uses __str__
v3 = v1 + v2 # Uses __add__
print(v3) # Vector(3, 7)
print(v1 == v2) # False - uses __eq__
print(len(v1)) # 2 - uses __len__

Class and Static Methods

Alternative method types in classes

Python classes support different types of methods:

  • Instance methods: Regular methods that operate on an instance (take self as first parameter)
  • Class methods: Methods that operate on the class itself (take cls as first parameter)
  • Static methods: Methods that don't operate on the instance or class (no special first parameter)
Instance Method

Works with instance data

Class Method

Works with class data

Static Method

Utility function in class namespace

# Class with different method types
class MyClass:
    class_attribute = "I'm a class attribute"

    def __init__(self, value):
        self.instance_attribute = value

    # Instance method
    def instance_method(self):
        return f"Instance value: {self.instance_attribute}"

    # Class method
    @classmethod
    def class_method(cls):
        return f"Class attribute: {cls.class_attribute}"

    # Static method
    @staticmethod
    def static_method(x, y):
        return f"Sum: {x + y}"

    # Alternative constructor using class method
    @classmethod
    def from_string(cls, string):
        value = int(string)
        return cls(value)

# Using the methods
obj1 = MyClass(10)
print(obj1.instance_method()) # Instance value: 10
print(MyClass.class_method()) # Class attribute: I'm a class attribute
print(MyClass.static_method(5, 3)) # Sum: 8

# Using alternative constructor
obj2 = MyClass.from_string("25")
print(obj2.instance_method()) # Instance value: 25

Property Decorators

Controlled attribute access

Property decorators allow you to define methods that can be accessed like attributes, providing a way to implement getters, setters, and deleters in a Pythonic way.

Benefits of using properties:

  • Validate data before setting attributes
  • Compute values on the fly
  • Maintain backward compatibility when changing implementation
  • Create read-only or write-only attributes
Pythonic Approach: Properties allow you to start with simple attributes and later add validation or computation without changing the interface.
# Class using property decorators
class Temperature:
    def __init__(self, celsius):
        self._celsius = celsius

    @property
    def celsius(self):
        return self._celsius

    @celsius.setter
    def celsius(self, value):
        if value < -273.15:
            raise ValueError("Temperature below absolute zero is not possible")
        self._celsius = value

    @property
    def fahrenheit(self):
        # Computed property
        return (self._celsius * 9/5) + 32

    @fahrenheit.setter
    def fahrenheit(self, value):
        # Set celsius based on fahrenheit value
        self._celsius = (value - 32) * 5/9

# Using the Temperature class
temp = Temperature(25)
print(temp.celsius) # 25 (getter)
print(temp.fahrenheit) # 77.0 (computed)

temp.celsius = 30 # Setter with validation
print(temp.fahrenheit) # 86.0

temp.fahrenheit = 100 # Set celsius via fahrenheit
print(temp.celsius) # 37.777...

# This would raise an error:
# temp.celsius = -300 # ValueError

OOP Best Practices

Writing clean and maintainable object-oriented code

Follow these best practices to write effective object-oriented code in Python:

  • Follow naming conventions: Use CamelCase for class names and snake_case for method/attribute names
  • Keep classes focused: Each class should have a single responsibility
  • Use composition over inheritance: Favor object composition over class inheritance when possible
  • Document your classes: Use docstrings to explain the purpose of your classes and methods
  • Use properties for computed attributes: Instead of getter/setter methods
  • Leverage polymorphism: Design interfaces that can work with multiple object types
Zen of Python: "Simple is better than complex" and "Flat is better than nested" - these principles apply to OOP design too.
# Example of good OOP practices
class BankAccount:
    """A simple bank account class.

    Attributes:
     account_holder (str): The name of the account holder
     balance (float): The current account balance
    """

    def __init__(self, account_holder, initial_balance=0):
        self.account_holder = account_holder
        self._balance = initial_balance

    @property
    def balance(self):
        """Get the current balance."""
        return self._balance

    def deposit(self, amount):
        """Deposit money into the account."""
        if amount <= 0:
            raise ValueError("Deposit amount must be positive")
        self._balance += amount
        return self._balance

    def withdraw(self, amount):
        """Withdraw money from the account."""
        if amount <= 0:
            raise ValueError("Withdrawal amount must be positive")
        if amount > self._balance:
            raise ValueError("Insufficient funds")
        self._balance -= amount
        return self._balance

    def __str__(self):
        return f"BankAccount(holder='{self.account_holder}', balance={self._balance})"

# Using the well-designed class
account = BankAccount("Alice", 1000)
print(account) # Clear string representation
account.deposit(500)
print(account.balance) # Using property
account.withdraw(200)
print(account) # Updated balance
